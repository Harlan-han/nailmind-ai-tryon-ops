# 页面信息架构

## 1. 文档目标
本文件定义甲感 NailMind 的页面层级、页面关系和主流程路径，用于后续原型设计与前端开发。

---

# 2. 用户端页面信息架构

## 2.1 页面树

### A. 首页 Home
- 产品价值区
- 立即试戴入口
- 热门风格入口
- 热门款列表
- 新品推荐
- 趋势标签区

### B. 上传手照页 Upload
- 上传区域
- 拍摄指引
- 示例图
- 上传确认

### C. 款式列表页 Style Gallery
- 风格筛选
- 场景筛选
- 款式卡片流
- 热门/新品标签
- 加入候选入口
- 试戴入口

### D. 试戴处理中页 Processing
- 处理状态
- 加载说明
- 返回上一步

### E. 试戴结果页 Try-On Result
- 原图 / 试戴图
- AI 适配说明
- 推荐款列表
- 收藏
- 加入候选
- 预约意向
- 再试一款

### F. 候选清单页 Candidate List
- 候选款列表
- 款式状态
- 删除候选
- 进入对比
- 继续试戴

### G. 多款对比页 Compare
- 2-4 款对比
- 差异说明
- AI 对比总结
- 最终选择

### H. 预约意向页 Booking
- 已选款式
- 试戴缩略图
- 门店/时间信息
- 提交预约意向

### I. 我的记录页 My
- 历史试戴
- 收藏列表
- 候选清单
- 最近推荐
- 偏好画像（P1）

---

## 2.2 用户端主流程

### 流程 1：首次体验主流程
首页 → 上传手照页 → 款式列表页 → 试戴处理中页 → 试戴结果页 → 预约意向页

### 流程 2：浏览后试戴
首页 → 款式列表页 → 上传手照页 → 试戴处理中页 → 试戴结果页

### 流程 3：多款决策
试戴结果页 → 候选清单页 → 多款对比页 → 预约意向页

### 流程 4：复访继续决策
我的记录页 → 历史试戴结果 → 收藏 / 再试 / 预约

---

## 2.3 用户端页面职责分工
- 首页负责引导
- 上传页负责保证试戴输入质量
- 列表页负责高效选款
- 结果页负责决策推进
- 候选与对比页负责缩小范围
- 预约页负责转化
- 我的页负责沉淀与复用

---

# 3. 运营端页面信息架构

## 3.1 页面树

### A. 总览页 Dashboard
- 今日核心指标
- 热门款榜
- 热门风格榜
- 异常提醒
- 日报入口

### B. 趋势分析页 Trends
- 时间筛选
- 趋势图
- 风格分布
- 漏斗分析
- 明细表

### C. 爆款候选页 Hot Candidates
- 候选榜
- 增速榜
- 稳定热门榜
- 建议动作

### D. 冷门预警页 Cold Warning
- 高曝光低试戴
- 高试戴低收藏
- 高收藏低预约
- 连续下滑款
- 优化建议

### E. 款式管理页 Style Admin
- 款式列表
- 上传款式
- 编辑标签
- 上下架管理
- 热门状态管理

### F. AI 日报页 Reports
- 今日日报
- 历史日报
- 复制导出
- 采纳标记

### G. 建议中心 Suggestions
- 建议列表
- 建议分类
- 建议详情
- 优先级
- 采纳状态

### H. 配置中心 Settings（P1）
- 标签配置
- 推荐规则配置
- 权重与策略配置

---

## 3.2 运营端主流程

### 流程 1：日常查看
总览页 → 趋势分析页 → 爆款候选页 → AI 日报页

### 流程 2：问题定位
总览页 → 冷门预警页 → 款式管理页 → 建议中心

### 流程 3：策略执行辅助
AI 日报页 → 建议中心 → 款式管理页 / 推荐位配置（后续）

---

## 3.3 运营端页面职责分工
- 总览页负责快速判断整体状态
- 趋势页负责解释变化
- 爆款页负责识别机会
- 冷门页负责识别损耗
- 管理页负责管理款式底层素材与标签
- 日报页负责生成日常行动结论
- 建议中心负责承接 AI 运营动作

# 6. 信息架构原则
1. 用户端围绕“试戴—决策—转化”展开
2. 运营端围绕“看趋势—找问题—拿建议”展开
3. 页面不是越多越好，而是要把闭环拆清楚
4. 所有页面设计都应服务于后续 AI 直接进入执行
